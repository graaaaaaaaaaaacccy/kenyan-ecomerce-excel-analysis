
Project Overview
This project analyzes simulated e-commerce transaction data for the Kenyan market in 2024. It explores customer behavior, product performance, payment preferences, and revenue distribution across major Kenyan cities — delivering actionable business insights through pivot tables and a KPI dashboard built entirely in Microsoft Excel.

 Business Questions Answered

Which city generates the most revenue?
What is the peak sales month of the year?
Which products and categories drive the most revenue?
What payment methods do Kenyan customers prefer?
How does order volume and revenue vary across the year?

Tools & Techniques Used

Microsoft Excel — core analysis and visualization tool
Pivot Tables — revenue aggregation by month, city, category, product, payment method
Formulas — SUMIF, COUNTIF, Revenue = Price × Quantity
KPI Dashboard — summary metrics and business insight cards
Data Relationships — manual relational structure across Customers, Products, and Orders

File Structure
Kenya_Ecommerce_Dataset.xlsx
├── Customers          # Dimension table: 50 customers across 5 cities
├── Products           # Dimension table: 30 products across 5 categories
├── Orders             # Fact table: 100 transactions in 2024
├── Pivot Table        # Revenue summary breakdowns
└── Dashboard          # KPI summary and key business insights

🚀 How to Use

Download and open the .xlsx file in Microsoft Excel (2016 or later recommended)
Start with the Dashboard sheet for a high-level overview
Explore the Pivot Table sheet to drill down by month, city, or category
Use the Orders, Customers, and Products sheets for raw data exploration
Refresh pivot tables: Data → Refresh All if you modify source data
